🔍 T001 首页技术体检结果（共 12 项发现）

来源：coordination/reports/seo-tech/T001-homepage-audit.md（v2，qa-reviewer 已批准） |  审查：coordination/reviews/T001.md（VERDICT: APPROVED，2026-06-18）
注意： "需改站走门禁"表示该修改必须经 advisor 向 Roger 说明并获确认后，由 devops 执行，不可自行操作。
# 问题描述 严重度 关键证据 建议动作 需改站走门禁
1 H1 标签出现 3 次（每页应只有 1 个）
谷歌规定每页只有一个最重要的大标题。现在三个"大标题"让谷歌搞不清首页重点		 高 H1[1]=ShowMo, H1[2]=Up to One Mile Wi-Fi Connectivity, H1[3]=WinEye 改主题模板代码（子主题），保留一个核心 H1 是
2 og:image 完全缺失
分享到微信/脸书/领英时无预览图，点击率明显降低		 高 抓取首页无任何 og:image meta 标签 Yoast 后台「社交→Facebook」上传品牌图 是
3 34 张图片均无 loading=lazy
一打开页面 34 张图同时下载，手机访问明显变慢		 高 loading="lazy" 出现次数=0；图片共 34 张 安装图片优化插件（如 EWWW Image Optimizer）或改主题代码 是
4 Title 格式含品牌冗余，关键词靠后
搜索结果蓝色标题未出现产品词，谷歌不知道你卖什么		 中 当前："Home - ShowMo - showmo365"（25字符） Yoast 后台改首页 SEO 标题，前置产品词（需 Roger 先确认关键词方向） 是
5 Twitter Card 不完整（仅有卡片类型，缺 title/description/image）
订了豪华相框但里面什么都没放		 中 仅见 twitter:card: summary_large_image，其余字段缺失 Yoast 后台「社交→Twitter」填写内容 是
6 首页无页面缓存（Cache-Control: no-cache）
每次访问服务器都重新生成页面，流量高峰时变慢		 中 响应头：cache-control: no-cache, must-revalidate, max-age=0；未检测到缓存插件指纹 安装 LiteSpeed Cache 或 W3 Total Cache（先确认是否已装） 是
7 <head> 内 2 个同步阻塞脚本（无 defer/async）
进门先等收银机开机再营业，拖慢首屏速度		 中 jQuery（ver=3.7.1）和 jQuery-migrate（ver=3.4.1）在 head 无 defer/async 改主题/插件代码加 defer 属性（需测试验证功能完整性） 是
8 sitemap 中含 /sample-page/（WordPress 默认示例页）
谷歌会白跑一趟爬这个无用页面		 低 page-sitemap.xml 中发现 /sample-page/ Yoast 后台将该页设为 noindex 或从 sitemap 排除 是
9 双规范 URL 风险：https://showmo365.com/ 可独立访问
两个地址都能打开，谷歌可能重复计算（canonical 已提供软规范，风险较低）		 中 实测（qa 复审确认）：https://showmo365.com/ 经 WordPress 层 301 跳 www，canonical 指向 www 版，风险低于 v2 原评估 服务器层（.htaccess）加 301 非www跳www（高风险操作，必须备份先行） 是
10 WebSite Schema description 字段为空字符串
递名片但背面空白，谷歌读不到网站自我介绍		 低 JSON-LD 中 "@type":"WebSite","description":"" Yoast 后台「常规」填写网站描述 是
11 WP 核心版本无法从公开页面确认
版本过旧可能有安全漏洞，需后台查看实际版本		 需确认 资源 URL 中多个 ?ver= 参数均为各组件版本，不能推断 WP 核心版本 devops 执行 wp core version（只读命令）确认版本 仅只读确认
12 Schema alternateName 含多品牌变体堆叠
Organization 含四个变体，存在轻微关键词堆叠风险		 低 Organization.alternateName="ShowMo showmo showmo365 ShowMo365" Yoast 后台精简，仅保留 1~2 个真实品牌名（需 Roger 确认哪些是官方别名） 是

A1 · 高优先级
修复首页三个"大标题"
谷歌规定每页只能有一个最重要的大标题。现在有三个，导致权重被分散。需改主题模板代码（中等风险），改前必须备份。
建议：现在做（最高优先级）
A2 · 高优先级 · 最容易
设置社交分享图片（og:image）
分享到微信/脸书时显示图片，点击率立竿见影。只需在 Yoast 后台点几下上传图片，不碰代码，风险极低。
建议：现在做（最容易的一项）
A3 · 高优先级
开启图片懒加载
34 张图片一次性全部加载导致网页变慢，手机用户影响更大。可安装图片优化插件一键开启（低风险），或改主题代码（中等风险）。改前必须备份。
建议：现在做（最高优先级）
A6 · 高优先级
安装页面缓存插件
服务器现在每次都重新生成页面，流量大时会变慢。安装 LiteSpeed Cache 即可大幅提速。安装前需先确认是否已有缓存插件（B2 项），避免重复安装。
建议：与 A3 一起处理
B1 · 只读确认
查清 WordPress 版本
devops 用一行只读命令查看 WP 版本，不改任何内容。版本过旧会有安全漏洞，必须先确认再决定是否升级。
建议：现在确认（不需备份）
B2 · 只读确认
确认是否已装缓存/图片优化插件
避免重复安装同类插件导致冲突。devops 查看已安装插件列表，10 分钟内完成，是 A3 和 A6 的决策前提。
建议：现在确认（和 A3/A6 一起决策）
A4 · 中等优先级
优化首页 Title 关键词
首页标题没有产品词，谷歌不知道你卖什么。需 Roger 先确认核心关键词方向（主打哪个产品词），确认后 devops 在 Yoast 后台改一行，风险极低。
建议：等 Roger 确认关键词方向后再做
